# PollScheduler 테스트 코드 안내

## 1. 사용된 테스트 기법

- **스파이 및 테스트 더블 (Spies & Test Doubles)**: Express의 Request, Response, Next 객체를 `jest.fn()`을 이용해 가짜 객체로 만들어 컨트롤러 함수에 주입했습니다.

## 2. 유사하거나 다른 기법 비교

- **E2E (End-to-End) 테스트**: `Supertest` 라이브러리를 사용해 가상의 Express 앱을 띄우고 HTTP 요청을 직접 보내서 응답을 검증하는 방식입니다.
- 컨트롤러 단위 테스트(현재 기법)는 프레임워크 런타임 없이 함수 호출만으로 검증하므로 훨씬 가볍습니다.

## 3. 이 기법을 도입한 이유

현재 `pollScheduler`는 오직 개발용으로 서버의 헬스체크 및 쿼리 동작을 검증하기 위한 엔드포인트입니다.
무거운 Express 애플리케이션 생명주기를 테스트에서 재현할 필요 없이,
Controller가 정상적으로 Repository를 호출하는지, 그리고 성공/실패 시 `res.status`나 `next()`를 올바르게 호출하는지(제어 흐름)만 검증하기 위해 이 방식을 채택했습니다.
